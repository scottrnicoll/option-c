Game Design Flow

TermDefinition
Game MechanicThe core gameplay pattern tied to a math domain. There are 19 mechanics total (e.g. "Balance & Equalize", "Split & Share"). Each has a stick figure animation. The mechanic determines HOW the player interacts with the math.
Game CardThe visual card the learner builds by making 5 choices. Contains: Theme, Character, Player Action, Win Condition, and Game Style. The Game Card defines WHAT the game looks and feels like.
Game Card ComponentsThe 5 slots on the Game Card:
1. Theme — the world/setting (e.g. "haunted castle")
2. Character — who the player is (e.g. "ghost hunter")
3. Player Action — what the player does on screen (e.g. "stack cursed bricks")
4. Win Condition — how the player wins (e.g. "complete 5 rounds")
5. Game Style — visual look (Cute, Stick Man, or Retro)
Math RoleAuto-filled from the Game Mechanic. Describes how math fits into the game (e.g. "Add and remove to make both sides match").
Game EngineThe pre-built game code for a mechanic. Handles all game logic (rounds, scoring, win/lose, difficulty). There are 19 engines (one per mechanic). The engine is the MACHINE that runs the game.
Theme ConfigA JSON object generated from the Game Card. Contains names, colors, and messages that skin the Game Engine. The config is the PAINT on the machine.
Game Style / VibeThe visual palette and font. Three options:
• Cute (soft pastels, chubby SVG characters, Quicksand font)
• Stick Man (dark zinc, white line art, Patrick Hand font)
• Retro (blue pixel art, blocky SVG, VT323 font)

Gameplay & Progress

TermDefinition
Moon StatusThe progression state of a skill:
• Locked (grey) — prerequisites not met
• Available (blue) — ready to start, at learner's grade
• In Progress (yellow) — learner started building a game
• In Review (yellow) — game submitted, waiting for guide
• Approved (yellow) — guide approved, needs to play 3x
• Demonstrated / Unlocked (green) — won own game 3x in a row
• Mastered (gold) — won 3 games by other learners

Hint CardPractice mode. When playing a game, learner can choose "Play with Hint Card" — a side panel shows the math concept, steps, and encouragement. Wins don't count toward mastery/tokens.
Game DareOptional challenge text the learner writes after building a game (e.g. "I dare you to beat this in under 30 seconds!"). Shown in the Game Library.

3. Game Engine Architecture

Games are built using pre-built game engines — one per Game Mechanic. The AI generates a Theme Config (names, colors, messages) but does NOT write game code. This guarantees games always work.

How it works

  • Learner fills the Game Card (Theme → Character → Action → Win → Style)
  • AI (Haiku) enhances each pick and generates a Theme Config JSON
  • Game Engine takes the Theme Config + math parameters and outputs complete HTML
  • HTML includes: game juice (particles, shake, combos), SVG characters, help button, round management, win/lose protocol

Engine Features (all engines)

  • 5 rounds with progressive difficulty
  • Randomized values every playthrough
  • Game juice: particle bursts, screen shake, combo counter, score pop-ups
  • SVG-drawn characters (no emoji)
  • Intro screen with "How to play"
  • Help button (?) with math explanation + valid/invalid examples
  • gameWin() / gameLose() protocol for parent app
  • Mouse, touch, and keyboard input
  • Three visual styles: Cute, Stick Man, Retro
