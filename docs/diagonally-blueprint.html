1. Glossary — The Galaxy

These terms describe the learner's map and how they navigate math.

TermDefinitionCommon Core equivalent
GalaxyThe 3D map of all math from K to 12. Contains 66 planets and 535 moons.—
PlanetA math domain at one grade level (e.g. "Geometry Grade 6"). Groups related skills. Shown as a glowing sphere in the galaxy.Domain
MoonA single math skill. One moon = one learning objective. Shown as a small dot orbiting a planet.Standard
Moon StatusThe progression state of a skill:
• Locked (grey) — prerequisites not met
• Available (blue) — ready to start, at learner's grade
• In Progress (yellow) — learner started building a game
• In Review (yellow) — game submitted, waiting for guide
• Approved (yellow) — guide approved, needs to play 3x
• Demonstrated (green) — won own game 3x in a row
• Mastered (gold) — won 3 games by other learners 		—
Mini-mapSmall overview panel showing all planets at a glance. Click a dot to fly to that planet.—
SearchSearch bar (top-left of galaxy) that finds moons by name, standard ID, or domain.—
Read Me!Info button that explains what the current screen element is.—

2. Glossary — Math & Game Mechanics

These terms describe how math is organized and turned into gameplay.

TermDefinition
Game MechanicThe core gameplay pattern tied to a math domain. There are 19 mechanics (e.g. "Balance & Equalize", "Split & Share"). Each has a stick figure animation that demonstrates the mechanic. The mechanic determines HOW the player interacts with the math.
Math RoleAuto-filled from the Game Mechanic. Describes how math fits into the game (e.g. "Add and remove to make both sides match"). The learner doesn't choose this — it comes from the mechanic.
Game EngineThe pre-built game code for a mechanic. Handles all game logic (rounds, scoring, win/lose, difficulty). There are 19 engines (one per mechanic). The engine is the MACHINE that runs the game. Engines are tested and guaranteed to work.
Game OptionA specific gameplay mode within a Game Engine. Each engine has 3 game options (57 total). Each option is a genuinely different way to play the same math — not just a difficulty tweak. Example: "Balance & Equalize" has Free Balance, Mystery Side, and Chain Scales. The learner picks one when building their game. Each game option card shows: the option name + a short description of what the player does.
Theme ConfigA JSON object generated by AI from the learner's Circuit Board Builder picks. Contains names, colors, messages, and sprite selections that skin the Game Engine. The config is the PAINT on the machine. AI generates it in ~2 seconds.
Sprite Library30 SVG assets used in games: 10 characters (pirate, robot, astronaut, knight, chef, diver, ghost, ninja, wizard, explorer), 10 items (coin, gem, treasure chest, crystal, potion, fruit, star, shell, mushroom, key), 10 backgrounds (underwater, space, forest, castle, kitchen, cave, city, volcano, arctic, jungle). Learner picks directly from grids in the Circuit Board Builder. Built by Mike.
Sprite PickerThe sprite grids inside the Circuit Board Builder. Learner clicks a thumbnail to select a background, character, or item from the library. No separate screen — sprites are picked inline as part of the builder. Sprite library built by Mike.
Round DataAI-generated math problems tailored to the specific Common Core standard. Each round has: a prompt (what the player sees), a target (correct answer), items (clickable numbers including answer + distractors), and a hint. Generated by Haiku at build time. Built by Mike.
Phaser EngineA next-generation game engine using the Phaser.js game framework (vs our original simple HTML engines). Currently only Collect & Manage uses Phaser. Supports sprite rendering, physics, and richer gameplay. Built by Mike.

Game Criteria

Every game must meet 3 criteria. Two are auto-checked when a mechanic is selected.

CriterionDefinitionWhen it lights up
🎮 Playable GameClear idea, clear goal, clear win/lose. A kid can understand and play it.When all 4 Circuit Board Builder slots are filled
🧠 Math Well AppliedThe math skill is used like in the real world — not simplified, not faked.Auto — when a Game Mechanic is selected
💎 Math EssentialKnowing the math skill is essential to win. Remove the math and the game breaks.Auto — when a Game Mechanic is selected

3. Glossary — Game Design & Visuals

These terms describe how the learner customizes what their game looks and feels like.

TermDefinition
Moon CardThe full-page view when you click a moon. Shows: Title, "This is about...", "Where you'll use this", collapsible "Learn more" (formula + watch out for), then "Build Your Game" button.
Circuit Board BuilderThe game builder screen. Learner picks components from a tray:
1. Math Role — auto-filled from the matched mechanic (not editable)
2. Background — pick from sprite library (10 backgrounds)
3. Character — pick from sprite library (10 characters)
4. Game Option — pick from 57 options, grouped by mechanic with expand/collapse
5. Items — pick from sprite library (10 items)

Game NamingAfter a game generates, the learner names it and writes an optional dare before testing it.
Game DareOptional challenge text (e.g. "I dare you to beat this in under 30 seconds!"). Shows in the Game Library when others play it.

4. Glossary — Gameplay & Progress

These terms describe how learners play, earn, and progress.

TermDefinition
TokensIn-app currency. Two balances tracked:
• Lifetime Tokens — total tokens ever earned. Never decreases. Used for leaderboard ranking.
• Spendable Tokens — available balance (will be spendable on future features).
Sources: +2000 per approved game, +100 per mastered skill, +10 per unique play of your game (all admin-configurable).
Hint CardPractice mode. When playing a game, the learner chooses "Play with Hint Card" — a side panel shows the math concept, steps, and encouragement. Wins don't count toward mastery/tokens. After winning, prompted: "Ready to play for real?"
Game LibraryAll approved games. Tabs: For My Grade, All Games, Ranking. Learners play others' games to master skills.
Ranking / LeaderboardTop learners by lifetime token count (not spendable balance). Top-3 podium + per-grade top 10.

• Diagonal Spark — a good, practical idea (1,000 tokens)
• Diagonal Idea — a really impactful idea (5,000 tokens)
• Diagonal Vision — a transformative idea (10,000 tokens)
Learners can win each level multiple times. All amounts admin-configurable.
Weekly Progress ChartLine graph showing skills demonstrated over time. Available on guide's Learners tab (all learners) and learner's My Stuff (self vs classmates).

7. All 19 Game Engines

Every Game Mechanic has a pre-built Game Engine with 3 game options each (57 total). The engine handles all game logic. AI only generates the Theme Config (names, colors, messages) — not game code. Generation takes ~3 seconds. See the full Game Options table below for all 57 gameplay modes.

Engine Features (all 19 engines share these)

  • 5 rounds with progressive difficulty
  • Randomized values every playthrough
  • Game juice: particle bursts (triple on victory), screen shake, combo counter, score pop-ups
  • SVG-drawn characters (no emoji)
  • Intro screen explaining how to play
  • Help button (?) with math explanation + valid/invalid examples
  • gameWin() / gameLose() protocol for parent app integration
  • Fail tracking: 3 wrong answers = game over (triggers Hint Card)
  • Mouse, touch, and keyboard input with smooth hover/drag feedback
  • Consistent visual style with Space Grotesk (titles) + Lexend (body) fonts
  • Theme-specific background colors generated by AI to match learner's chosen theme

What's hardcoded vs what's AI

This is important to understand: the game logic is built by us, not by AI. AI only handles the creative "skin" on top.

PartWho builds itWhat it means
Game logic (rounds, scoring, win/lose)Hardcoded (us)Every engine has the same structure: 5 rounds, progressive difficulty, randomized numbers. This never changes and always works.
Win conditionHardcoded (us)Win = complete all 5 rounds. Lose = 3 wrong answers in a round. The learner's win condition text (e.g. "complete 5 rounds before time runs out") appears in the intro and help panel, but the actual game logic is the same for all games using that engine.
Game juice (particles, shake, combos)Hardcoded (us)Every game has the same celebration effects, screen shake on mistakes, and combo counter. Built into the shared base template.
Win/lose detectionHardcoded (us)gameWin() and gameLose() are called by the engine code. The parent app listens for these to track mastery, show the Hint Card, and count wins.
Fail trackingHardcoded (us)3 wrong answers in a round = game over. Resets each round. Triggers the Hint Card / Math Moment when the learner loses.
Game title and namesAI (Haiku)The learner's Circuit Board Builder picks are sent to AI, which generates short themed names for the title, character, items, world, and win/lose messages. Takes ~2 seconds.
Background colorsAI (Haiku)AI picks background colors that match the learner's chosen background and character (e.g. deep blue for underwater). Falls back to a default palette if AI fails.
Concept explanationsAI (Sonnet, first time) then cachedThe "What is this?" explanation on each moon is generated once by AI, then cached in Firestore forever. After the first visit, it loads instantly.
Hint Card contentAI (Haiku, first time) then cachedThe math help shown when a learner loses is generated once, then cached. After the first loss on that concept, it loads instantly.
Sprite selectionLearnerLearner picks background, character, and item sprites directly from the library grids in the Circuit Board Builder. No AI involvement — the learner chooses freely. Library built by Mike.
Math round problemsAI (Haiku, per game)For the Phaser engine, AI generates 5 rounds of math problems tailored to the specific Common Core standard. Not generic math — tests the EXACT skill. Built by Mike.

Bottom line: The game itself (how you play, how you win, how you lose) is 100% built by us and always works. The learner picks sprites (background, character, items) and game option. AI generates themed names, colors, and math problems matched to the specific standard.

8. How Game Generation Works

When a learner builds a game, here's what happens step by step:

StepWhat happensWho does itTime
1. Pick a moonLearner clicks a blue moon on the galaxy or searches for a skillLearner—
2. Read the conceptMoon Card shows: "This is about...", "Where you'll use this", collapsible "Learn more" (formula + watch out for)Cached / AI (first time only)Instant (cached) or ~2s
3. Open builderLearner clicks "Build Your Game" → full-page Circuit Board Builder opensLearner—
4. Pick componentsLearner selects 4 components from the builder tray: Background, Character, Game Option (grouped by mechanic), Items. Math Role is auto-filled.Learner—
5. Build the gameGame Engine generates HTML from the Theme Config. AI generates themed names/colors based on the learner's picks. No AI writes game code.Engine + AI (Haiku for config)~3 seconds
6. Name itLearner names their game and writes an optional dare.Learner—
7. Test & submitLearner plays the game in the Workshop. Must win at least once before submitting for review.Learner—
8. Guide reviewGuide plays the game, then approves or sends back with feedback ("Needs Fix").Guide—
9. DemonstrateAfter approval, learner must win their own game 3 times in a row → moon turns green.Learner—
10. MasterPlay and win 3 games by OTHER learners on the same skill → moon turns gold.Learner—
